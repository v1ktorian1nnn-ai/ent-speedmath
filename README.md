# ENTRace — соревнования по скорости решения задач профильной математики ЕНТ

Полноценный сайт: регистрация/вход, тренировка на время с общим рейтингом,
дуэли 1×1 и групповые гонки в реальном времени, история результатов,
переключатель языка RU/KZ, админка для загрузки своего банка задач
(текст + картинки).

Стек:
- **Backend**: Node.js, Express, Socket.io (реал-тайм дуэли), Prisma + PostgreSQL, JWT.
- **Frontend**: React (Vite), react-router, socket.io-client, обычный CSS (без сборщика типа Tailwind — проще разворачивать).

## 1. Локальный запуск

### 1.1 База данных

Нужен PostgreSQL. Проще всего — бесплатная база на [Neon](https://neon.tech)
(регистрация, "New Project", скопировать `DATABASE_URL` из Connection Details).
Либо локальный Postgres/Docker, если он у тебя уже есть.

### 1.2 Backend

```bash
cd backend
cp .env.example .env
# впиши в .env свой DATABASE_URL и любой случайный JWT_SECRET

npm install
npx prisma migrate dev --name init   # создаст таблицы
npm run seed                          # тестового админа и пример задач
npm run dev                           # старт на http://localhost:4000
```

Тестовый админ после `npm run seed`: `admin@example.com` / `admin12345`
(смени пароль в БД или через будущую форму смены пароля — сейчас её нет,
это самый простой первый шаг для доработки).

### 1.3 Frontend

```bash
cd frontend
cp .env.example .env   # по умолчанию указывает на http://localhost:4000
npm install
npm run dev             # старт на http://localhost:5173
```

Открой `http://localhost:5173`.

## 2. Как наполнить банк задач своими данными

Зайди под аккаунтом администратора → раздел "Админка" (появляется в меню
только у пользователей с `isAdmin = true`). Там форма: тема, сложность,
текст условия, варианты ответа через запятую, индекс правильного варианта
(считая с 0), необязательная картинка. Картинки сохраняются на диск сервера
в папку `backend/uploads` и раздаются по `/uploads/...`.

Если задач много — быстрее массово загрузить их напрямую в БД одним скриптом
по образцу `backend/prisma/seed.js`, а картинки заранее сложить в `backend/uploads`.

## 3. Бесплатный/дешёвый деплой

Рекомендуемая простая связка — **Render** (backend, Node-процесс с WebSocket)
+ **Neon** (Postgres) + **Render Static Site** или **Vercel** (frontend).

### 3.1 База — Neon
Уже сделано на шаге 1.1, если делал локальный запуск. Просто используй тот
же `DATABASE_URL` в проде.

### 3.2 Backend — Render
1. Зарегистрируйся на [render.com], "New" → "Web Service", подключи репозиторий с этим кодом (нужно сначала запушить проект на GitHub).
2. Root directory: `backend`.
3. Build command: `npm install && npx prisma generate && npx prisma migrate deploy`.
4. Start command: `npm start`.
5. Environment: добавь `DATABASE_URL`, `JWT_SECRET`, `FRONTEND_ORIGIN` (позже впишешь туда адрес фронтенда), `PORT` можно не указывать — Render сам подставит.
6. Бесплатный план Render "засыпает" после 15 минут простоя и просыпается ~30-50 секунд на первый запрос — это нормально для пет-проекта, но для реальной нагрузки в день экзаменов стоит перейти на платный план.

### 3.3 Frontend — Render Static Site или Vercel
1. Root directory: `frontend`. Build command: `npm run build`. Publish directory: `dist`.
2. Environment variable: `VITE_API_URL=https://<твой-backend>.onrender.com`.
3. После деплоя фронтенда вернись в настройки backend на Render и пропиши `FRONTEND_ORIGIN=https://<твой-frontend-адрес>` (иначе CORS заблокирует запросы).

### 3.4 Домен
И Render, и Vercel дают бесплатный поддомен (`*.onrender.com`, `*.vercel.app`) —
для старта этого достаточно, свой домен можно подключить позже в настройках.

## 4. Известные упрощения текущей версии (что стоит доработать дальше)

- Изображения задач хранятся на локальном диске backend-сервера. На Render
  файловая система эфемерна (сбрасывается при передеплое) — для продакшена
  стоит переехать на S3-совместимое хранилище (например, Cloudflare R2,
  бесплатный тариф достаточно щедрый).
- Групповая дуэль стартует по таймеру ожидания (15 сек) либо при наборе
  максимума игроков — нет ручного лобби с видимым списком участников.
  Это осознанное упрощение MVP.
- Нет восстановления пароля по email (нет настроенной почтовой отправки).
- Нет модерации/пагинации в админке — ок для банка на сотни задач, для
  тысяч стоит добавить пагинацию и поиск.
